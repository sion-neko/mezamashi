# 18_mezamashi

Expo (React Native) で作った目覚ましアラームアプリ。ローカル通知でアラームを鳴らし、AsyncStorage にセット状態を保存する。

## 機能

- 時刻ピッカーでアラーム時刻をセット
- 「毎日繰り返す」トグル
- アプリを閉じていてもローカル通知でアラームが鳴る（起床時刻から2秒間隔・60件のバースト通知）
- アラーム画面で「止める」「解除」操作
- アプリ再起動時もセット状態を復元（AsyncStorage）

## 必要環境

- Node.js（LTS推奨）
- スマホ実機の [Expo Go](https://expo.dev/go) アプリ
  - 本プロジェクトは **Expo SDK 54** を使用（`package.json` の `expo: ^54.0.0`）
  - Expo Go は **SDK バージョンが一致している必要がある**。手元の Expo Go が `54.0.2` であれば SDK 54 系なので対応OK
  - Expo Go のバージョンが古い/新しい場合はストアで最新版に更新してから使う

## セットアップ

```bash
npm install
```

## 起動手順

```bash
npm start
```

ターミナルに表示される QR コードを、スマホの Expo Go アプリ（iOSはExpo Go内のカメラ、Androidは標準カメラでも可）で読み取ると実機で起動する。

その他の起動方法:

```bash
npm run android   # Androidエミュレータ/実機
npm run ios       # iOSシミュレータ（Macのみ）
npm run web       # ブラウザ
```

## 使い方

1. 起動直後の画面で時刻をセットし「セット」を押す
2. 初回は通知の許可を求められるので許可する（拒否するとアラームが鳴らない）
3. セット後は「armed」画面でカウントダウン表示。「解除」で取り消せる
4. 時刻になると通知が届き、アプリを開くとアラーム画面（赤背景）が表示される。「止める」を押すと停止（毎日繰り返しの場合は次の発火時刻を再セット）

## ディレクトリ構成

```
App.tsx        画面・状態遷移（idle / armed / ringing / expired）
src/alarm.ts   アラームのロジック（保存・復元、通知スケジュール、次回発火時刻の計算）
index.ts       エントリーポイント
app.json       Expo設定
```

## 注意点

- iOSのローカル通知は同時予約数の上限があるため、バースト通知は60件（2秒間隔=約2分間）に制限している（`src/alarm.ts` の `BURST_COUNT`）
- 通知権限が許可されていない状態では、アラームをセットしても鳴らない
